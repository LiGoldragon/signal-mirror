# Skill — signal-mirror

*Working effectively in the payload-blind mirror signal contract.*

## What this repo owns

`signal-mirror` owns the ordinary working wire vocabulary for the
version-control mirror triad. Component-side shippers and restorers use
these types to append opaque versioned-log entries, publish checkpoints,
notify object heads, restore data, and observe mirror heads.

The repo does not own runtime behavior. Validation, durable state, socket
listeners, authentication, and append decisions live in the `mirror` daemon
or in `meta-signal-mirror`.

## Working rules

- Read `INTENT.md`, `AGENTS.md`, and `ARCHITECTURE.md` before editing.
- Edit `schema/lib.schema`, then regenerate with
  `SIGNAL_MIRROR_UPDATE_SCHEMA_ARTIFACTS=1 cargo build`.
- Do not hand-edit `src/schema/lib.rs`; it is generated by
  `schema-rust`.
- Keep this contract payload-blind. Entry and checkpoint records carry chain
  metadata beside opaque bytes; no component record type appears here.
- Keep digest wire forms fixed-width blake3 bytes.
- Keep dependencies as remote git dependencies in `Cargo.toml`; no sibling
  `path = "../..."` dependencies or local flake overrides.
- Preserve the round-trip witnesses in `tests/round_trip.rs` for both rkyv
  frames and NOTA text.

## Schema notes

Struct fields use the current positional schema syntax. Bare type references
derive their field role from the type. Explicit roles use the dot form for
plain references and the parenthesized form for composite references.

This is a `WireContract` crate: generated code should expose wire types and
codecs only, with no actor runtime or engine traits.
